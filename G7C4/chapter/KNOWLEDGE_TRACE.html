How to read the Knowledge Trace

The trace is a reviewer audit ledger. Each row is one module (or applet, indented) after A2/A3 review. Click Show details on a row to see every column the reviewer can inspect.

Snapshot row

  • Module id — e.g. G7C4M1. A purple version badge like v2 (1 archived) means prior versions of this module have been archived to KNOWLEDGE_TRACE_HISTORY.jsonl after a rework.
  • Verdict (ship / upgrade / rework / fail) + overall score percentage.
  • LO badge (LO Y / LO partial / LO N) — summary of LO coverage from the mastery scorer, with coverage percent.
  • must-fix / should-fix counts — aggregated carry-forward items from the scorer's slide actions.
  • Quadrant bars — P / D / X / L quadrant scores (green ≥ 80%, amber 60-80%, red below).

Ledger groups

LO & concepts
What the LO actually covered, which concepts were taught with slide evidence, which prior concepts were reactivated (reinforced), and LO scope mismatches (gaps and creep).
Vocabulary & language
Every term the module was expected to handle. The Source column distinguishes where the term came from: A1 plan = from the chapter module sequence (authoritative), KB = supplementary term from the knowledge base, scorer = mastery-discovered on-slide, carried = reused from a prior module. The Definition / Observed cell shows the authoritative A1 definition and (below, smaller) what the scorer actually observed the term doing on slides. Notation and formulas follow the same pattern. Reused from links to the prior module the term came from, with a drift warning if definitions don't line up.
Visual models & representations
Visual Insight Records (VIRs) used on slides, with their CPA role (concrete / pictorial / abstract). C/P/A distribution shows how many slides land in each phase. IAS archetypes are the interaction patterns the module employed.
Misconceptions
Addressed shows misconceptions the module surfaced, each with an effectiveness score 0-3 (0 = not addressed, 1 = partially, 2 = shown, 3 = tested). Missed shows misconceptions A1 listed but the module never touched, with severity.
Prerequisites & recalls
Prerequisites from the research brief and whether the module activated them (all / partial / none). In-chapter and out-of-chapter recalls flag moments where the module reactivated knowledge from earlier modules or chapters.
Progression
Depth progression shows concept confidence transitions across modules (e.g. variable: mentioned → established (prior: G7C4M1)). Exit verification lists any Exit Question trope positions and what they check.
Textbook & carry-forward
Textbook coverage shows the relationship between this module and its textbook references. A green delivered badge means A1 planned the reference AND the module's slides actually used it. A red gap badge means A1 planned it but no slide referenced it. An amber unplanned bonus badge means the module referenced a textbook page A1 didn't list. (The direction is always textbook → module: does this module deliver on its planned textbook targets?)

Carry forward is the scorer's prioritized slide actions (must-fix / should-fix) + research gaps — the concrete actions that should flow to the next rework or module.

Reviewer notes are free-text directives from approved gates.
Scoring signals
Gates failed — G1-G8 with descriptions on hover. Antipatterns — AP1-AP9 with descriptions.

The renderer never adds data the stored trace doesn't already have, so downstream consumers (next-module reviewers, batch dashboards, exports) see exactly the same facts you see here.
